# Surface Visual Regression Evidence

Status: experimental Block-system production-hardening evidence.

Surface visual regression reports use schema
`tetra.surface.visual-regression.v1` and level
`surface-visual-golden-v1`. The report is generated by
`tools/cmd/surface-golden`, validated by
`tools/cmd/validate-surface-visual-report`, and gated by
`scripts/release/surface/visual-gate.sh`.

The required scene set is:

- `command-palette`
- `dashboard`
- `settings`
- `editor`
- `glass`

Each scene stores the source scene path, source scene SHA-256, target,
`software-rgba` renderer, renderer version, dimensions, stride, current and
baseline RGBA frame SHA-256 checksums, baseline/current/diff PNG artifact
paths, PNG SHA-256 hashes, font manifest hash, asset manifest hash, and
zero-tolerance diff result. The canonical source scenes live under
`testdata/surface-golden/scenes/`.

The validator rejects fake golden evidence:

- screenshot-only evidence without a source scene hash;
- missing baseline/current/diff PNG artifacts;
- tampered artifact checksum or size;
- changed current-vs-baseline frame or PNG hash without the
  `surface-visual-review-approved` review marker;
- missing renderer version;
- missing font or asset manifest hashes.

Visual regression evidence is not Electron or Chromium pixel parity, CSS
browser rendering parity, or GPU compositor parity. It is a deterministic guard
that prevents Surface UI regressions from hiding inside the scoped Block/Morph
software RGBA path.
