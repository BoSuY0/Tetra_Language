# P20.0 Benchmark Matrix Hardening Foundation Design

Status: local foundation design for the active Ideal Master Plan goal loop.

## Scope

This slice hardens the benchmark matrix contract before any performance claims.
It extends the existing `truth-bench-harness` rather than replacing P8/P19
artifacts.

Covered evidence:

- a named `p20.0_benchmark_matrix` scope
- the P20.0 master-plan category list
- Tetra, C `clang -O3`, C++ `clang++ -O3`, and Rust
  `rustc -C opt-level=3` rows for every category
- matching `algorithm_id` and `input_description` across languages for each
  category
- host and per-row target CPU evidence
- compiler version and exact build/run commands for every row
- archived raw output artifact paths for every row
- Tetra proof, allocation, bounds, and performance report artifacts
- claim-policy validation that still forbids broad fastest-language,
  C++/Rust parity, official benchmark, and production database claims

## Non-Claims

- no measured performance comparison
- no C/C++/Rust parity
- no official benchmark result
- no TechEmpower publication
- no production database benchmark
- no P20.1 performance-blocker completeness
- no P20.2 claim-tier promotion

## Implementation Shape

Add `scopeP20BenchmarkMatrix = "p20.0_benchmark_matrix"` to
`tools/cmd/truth-bench-harness`.

Add a P20 policy with exact master-plan categories:

- integer loops
- slice sum
- bounds-check loops
- function calls
- recursion
- matrix multiply
- hash table
- allocation
- region/island allocation
- JSON parse/stringify
- HTTP plaintext/json
- PostgreSQL single/multiple/update
- actor ping-pong
- parallel map/reduce
- startup time
- binary size
- compile time

The policy requires equivalence metadata, raw output artifacts, and Tetra
performance report artifacts. Report validation also rejects row target CPU
values that do not match the captured host target CPU.

Keep existing `p8_full` and P19 scoped behavior compatible.
