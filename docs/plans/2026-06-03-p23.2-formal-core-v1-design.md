# P23.2 Formal Core v1 Design Bridge

Status: approved by active Ideal Master Plan goal loop.

Canonical plan: `/home/tetra/Downloads/tetra_ideal_master_plan_20260602.md`.

## Observed Facts

- `compiler/internal/formalcore` already exposes `tetra.formal_core.v1`
  minimum spec validation for values, provenance, borrow/copy, bounds proofs,
  allocation intent, and check-elimination validity.
- `compiler/internal/validation.CheckBoundsProofsWithPLIR` machine-checks
  removed bounds checks against live PLIR proof guards.
- `compiler/internal/validation.ValidateAllocationLowering` machine-checks
  allocation plans against lowered IR, including stack and function-temp region
  lowering paths.
- `compiler/internal/plir.VerifyProgram` rejects inconsistent provenance,
  borrow/copy, region, no-alias, and allocation-intent facts.
- `compiler/internal/runtimeabi` already records raw pointer bounds metadata for
  allocation-base roots, derived offsets, checked external/unknown pointers,
  and rejected impossible pointer arithmetic.

## Design

P23.2 adds a top-level compiler evidence report with schema
`tetra.formal_core.v1` and scope `p23.2_formal_core_v1`. The report does not
change runtime behavior. It connects the existing internal formal-core package
to live witnesses for the current supported subset.

Rows:

- values;
- borrows and owned/copy;
- provenance and regions;
- bounds proof id semantics;
- allocation length contract;
- allocation intent lowering;
- raw pointer bounds metadata;
- check-elimination validity.

The report validator rejects missing rows, missing witnesses, placeholders, and
fake claims for a full formal language proof, broad language proof, unsafe
policy change, runtime behavior change, safe-semantics change, or performance.

## Implementation Shape

- Extend `compiler/internal/formalcore.MinimumSpec()` with explicit P23.2
  concepts for regions, raw pointer bounds metadata, and allocation length
  contract.
- Add `compiler/formal_core_v1.go` with `BuildP23FormalCoreV1Report()` and
  `ValidateP23FormalCoreV1Report()`.
- Reuse existing live validators instead of introducing a second proof engine:
  `formalcore.ValidateSpec`, `plir.VerifyProgram`,
  `validation.CheckBoundsProofsWithPLIR`,
  `validation.ValidateAllocationLowering`,
  `runtimeabi.NewRawAllocationBounds`,
  `runtimeabi.DeriveRawPointerBounds`, and
  `runtimeabi.RawSliceBoundsFromParts`.
- Link the report through the verified-track registry, audit docs,
  `current_supported_surface`, `truthful_intent_architecture`,
  `explainable_one_build`, and `docs/generated/manifest.json`.

## Non-claims

- No full formal proof of Tetra is claimed.
- No broad language theorem prover is claimed.
- No public source interpreter or backend selector is introduced.
- Unsafe policy does not change.
- Runtime behavior does not change.
- Safe-program semantics do not change.
- No performance claim is made.
