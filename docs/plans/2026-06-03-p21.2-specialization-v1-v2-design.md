# P21.2 Specialization v1/v2 Design

Date: 2026-06-03

## Goal

Close the Ideal Master Plan P21.2 slice with evidence that selected
high-level abstractions disappear before machine code when types and targets
are known:

- generics;
- protocol/static conformance;
- extension methods;
- enum match known cases;
- optionals;
- collections.

## Navigation

Graphify MCP was used first. Its semantic search surfaced nearby protocol
conformance and machine IR nodes, but not a precise `InliningSpecialization`
node. Concrete inspection selected these sources:

- `compiler/internal/opt/coverage.go` for P17.2 specialization rows.
- `compiler/internal/opt/inlining.go` for `inline-small-pure`.
- `compiler/internal/opt/sccp.go` for known tag/presence branch folding.
- `compiler/internal/machine/scalar.go` for verified Stack IR to Machine IR
  scalar lowering.
- `compiler/internal/stdlibrt/stable_generic_collections.go` and
  `lib/core/collections.tetra` for P19.1 generic collection boundaries.

## Design

Add a new report in `compiler/internal/opt`:

- schema: `tetra.optimizer.specialization_machine_code.v1`;
- scope: `p21.2_specialization_v1_v2`;
- rows: one row per P21.2 target;
- each row records source evidence, optimized Stack IR evidence, Machine IR
  evidence, removed high-level markers, pass owners, and non-claim boundaries;
- a machine witness builds a tiny known direct-call program, runs
  `inline-small-pure`, verifies the optimized Stack IR no longer contains
  `IRCall`, lowers the optimized main function through
  `machine.ScalarIntFunctionFromStackIR`, verifies it, and records that the
  Machine IR contains no `OpCall`;
- known enum/optional rows tie to SCCP branch-fold evidence and the same
  machine witness as a narrow proof that folded high-level branch abstractions
  do not require machine-level dispatch/calls in the accepted scalar path;
- collection evidence ties P19.1 source monomorphization to the same bounded
  direct-helper witness only. It does not promote allocator-backed
  `Vec<T>`/`HashMap<K,V>` runtime behavior.

## Validator

`ValidateSpecializationMachineCodeCoverage` rejects:

- wrong schema or scope;
- missing or duplicate target rows;
- placeholder/missing evidence;
- missing optimized IR or Machine IR evidence;
- missing non-claims;
- fake broad specialization;
- fake dynamic dispatch or witness-table behavior;
- fake runtime generic value support;
- fake broad/allocator-backed collection runtime;
- fake layout/ABI freedom;
- fake performance claim;
- fake safe-semantics change.

## Non-Claims

P21.2 is evidence/report coverage only. It does not add a public optimizer
mode, a runtime behavior change, broad optimizer completeness, dynamic
protocol dispatch, runtime generic values, allocator-backed generic
collections, layout/ABI freedom, or performance evidence.
