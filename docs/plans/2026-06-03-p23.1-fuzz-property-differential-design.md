# P23.1 Fuzz / Property / Differential Expansion Design

Status: bridge before RED/GREEN.

## Goal

Close P23.1 as an evidence-backed fuzz/property/differential expansion for the
currently supported compiler/runtime subset. The slice records a machine-checked
coverage ledger with live witnesses; it does not add a public compiler mode or
claim exhaustive fuzzing.

Schema: `tetra.fuzz.property.differential.v1`

Scope: `p23.1_fuzz_property_differential`

## Evidence Sources

- `compiler/tests/fuzz` and a new P23.1 generated-source witness for parser,
  checker, lowering, PLIR, and Stack IR verification.
- `compiler/internal/differential.CheckBackendMatrix` for source, Stack IR,
  optimized Stack IR, SSA, Machine IR, randomized samples, first-mismatch
  reducer metadata, and host-supported native execution.
- `scripts/dev/fuzz-nightly.sh`, `docs/testing/fuzz_property_stress.md`, and
  `tools/cmd/validate-fuzz-summary` for the bounded nightly/short fuzz gate and
  artifact shape.
- `compiler/internal/runtimeabi` for deterministic allocator alignment and
  invalid-size property checks.
- `compiler/internal/actorsafety.TypedActorOwnershipTransferCoverage` for actor
  transfer stress/diagnostic boundaries.

## Coverage Rows

- `parser_checker_generated_programs`: generated source snippets parse and
  check through the public compiler API.
- `plir_lowering_verifier_pipeline`: the same snippets build PLIR, lower to
  Stack IR, and pass `VerifyIRProgram`.
- `backend_differential_matrix_expansion`: backend matrix compares source,
  Stack IR, optimized Stack IR, SSA, Machine IR, and randomized samples.
- `native_backend_boundary`: host-supported Linux x64 native lane is compared
  when available; other hosts record an explicit unavailable boundary.
- `runtime_allocator_properties`: region allocator alignment accepts valid sizes
  and rejects negative/overflow sizes.
- `actor_transfer_stress_boundary`: actor ownership transfer coverage includes
  stress diagnostics and PLIR moved facts without promoting distributed
  zero-copy.
- `fuzz_nightly_summary_gate`: the nightly wrapper, docs, and summary validator
  define required artifacts, logs, unstable seeds, and crasher inventory.
- `reducer_failure_artifacts`: mismatch evidence records a reduced single-sample
  reproducer.

## Non-claims

- No full program correctness claim.
- No exhaustive fuzzing claim.
- No full native differential suite for every target.
- No broad random program generator beyond the bounded snippets and existing Go
  fuzz targets in this slice.
- No performance claim.
- No runtime behavior change.
- No safe-program semantics change.
