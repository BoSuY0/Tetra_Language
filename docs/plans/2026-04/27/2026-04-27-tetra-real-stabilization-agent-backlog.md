# Tetra Real Stabilization Agent Backlog

**Status:** active replacement for the mechanical 5000-checkbox stabilization list.
**Created:** 2026-04-27
**Purpose:** give subagents real engineering work: investigate, fix, refactor, add tests, add
examples, harden gates, improve docs, and report evidence.
**Rule:** no task may be closed by simply rerunning an existing green command. Each task requires at
least one of: a new regression test, a real code/doc/script change, a removed ambiguity, a tighter
validator, a reduced flaky path, or a documented post-v1 decision backed by repo evidence.

## How Agents Must Work

- Start each task by reading the listed files and confirming the actual current behavior.
- Prefer a failing or missing-coverage test first, then implementation, then focused verification.
- Keep each agent in one domain unless explicitly reassigned.
- Do not edit this file until the task is genuinely completed and evidence is available.
- Do not update generated baselines unless the task explicitly asks for reviewed baseline work.
- Every final report must include changed files, commands run, observed results, remaining risks,
  and the exact task IDs completed.
