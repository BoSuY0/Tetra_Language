# P20.2 Claim Tiers Design

Status: design bridge before RED/GREEN.

## Observed Control Point

Graphify MCP and concrete inspection point at
`tools/cmd/truth-bench-harness/main.go` as the narrowest claim-policy control
point. `validateReport` already calls `validateClaims`, and the existing tests
reject broad fastest-language, fake C++/Rust parity, fake official benchmark,
fake official TechEmpower, and production database benchmark wording.

P20.0 and P20.1 currently provide only local dry-run matrix and blocker
explanation evidence:

- `p20.0_benchmark_matrix` proves the matrix/evidence contract with
  `ran=false`.
- `.perf.json` schema version 3 explains P20.0 Tetra benchmark blockers.
- Neither artifact proves measured speed, parity, official results, or
  cross-machine reproduction.

## Tier Policy

P20.2 adds an explicit performance claim-tier report and validator:

- Tier 0: local smoke only.
- Tier 1: local benchmark evidence.
- Tier 2: reproducible cross-machine benchmark.
- Tier 3: independent reproduced benchmark.
- Tier 4: official upstream benchmark submission.

The current repository evidence may emit only Tier 0 public wording for P20.0
and P20.1. Tier 1 through Tier 4 are policy rows for future promotion and must
require matching evidence classes before their wording is allowed.

## Validation Rules

The validator should reject:

- unknown or missing tiers;
- missing required evidence classes for a declared tier;
- placeholder evidence or wording;
- Tier 1 measured/local benchmark wording without local benchmark evidence;
- Tier 2 cross-machine reproducibility wording without cross-machine evidence;
- Tier 3 independent reproduction wording without independent evidence;
- Tier 4 official/upstream/TechEmpower wording without official upstream
  submission evidence;
- legacy broad claims such as fastest-language, C++/Rust parity, measured
  speed advantage, and official benchmark wording unless they are explicit
  non-claims.

The unstructured `Report.Claims` validator should also reject fake Tier 2,
Tier 3, and Tier 4 wording. Row-specific local measurements remain allowed only
when the text cites a benchmark row, schema/report, target, runtime, and
comparison row.

## Artifacts

The slice should add a checked audit/doc link and a generated policy artifact:

- `docs/audits/claim-tiers-v1.md`
- `reports/claim-tiers-v1/claim-tier-report.json`
- feature registry and manifest entries pointing at the audit and benchmark
  harness docs

This is a wording/evidence contract only. It must not run benchmarks, promote
P20.0/P20.1 dry-run evidence to measured performance, change optimizer/runtime
behavior, or alter safe-program semantics.
