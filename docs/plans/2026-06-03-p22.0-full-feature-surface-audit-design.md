# P22.0 Full Feature Surface Audit Design

Date: 2026-06-03
Slice: P22.0 Full Feature Surface Audit

## Intent

P22.0 records the current language/product feature surface before the Phase K
promotion work begins. The audit must decide which surfaces are already current
only within their bounded registry scope, which remain experimental/planned, and
which stay post-v1 until same-branch evidence exists.

This is an evidence/reporting slice. It must not promote P22.1 first-class
callable work, P22.2 protocol/trait-object work, runtime generic values,
structured concurrency, macro systems, cross-platform native UI, distributed
EcoNet, or proof-carrying capsules.

## Evidence Sources

- `FeatureRegistry()` is the canonical current/planned/post-v1 status source.
- `GetManifest()` must continue exporting the feature registry into
  `docs/generated/manifest.json`.
- `docs/spec/v1_scope.md` defines the future v1 release contract and the
  explicit post-v1 list.
- `docs/spec/v1_feature_status.md` records planning decisions without
  overriding registry status.
- `docs/spec/current_supported_surface.md`,
  `docs/design/explainable_one_build.md`, and
  `docs/design/truthful_intent_architecture.md` are the public non-claim docs
  that should point at the P22.0 audit.

## Report Shape

Add a compiler-level report:

- schema: `tetra.language.feature_surface_audit.v1`
- scope: `p22.0_full_feature_surface_audit`
- category rows for:
  - first-class callables
  - closures
  - protocols / trait objects
  - runtime generics
  - advanced enums / pattern matching
  - async typed errors
  - structured concurrency
  - modules / packages
  - macros / metaprogramming
  - UI / surface
  - Eco / capsules

Each row records:

- current decision
- feature IDs inspected
- registry statuses copied from `FeatureRegistry()`
- same-branch evidence strings
- future or post-v1 boundaries
- required promotion evidence

Rows must reject placeholder evidence and unknown or drifted feature IDs.

## Guardrails

The validator must reject:

- missing or duplicate categories
- fake same-branch promotions
- missing registry-backed feature IDs, except the macro/metaprogramming row,
  which may explicitly record the absence of a current macro feature
- placeholder evidence
- claims for full v1 guarantees, runtime generic values, trait objects,
  macro systems, full structured concurrency, cross-platform production UI,
  distributed EcoNet, proof-carrying capsules, broad performance, or safe
  semantics changes

## Expected Artifacts

- compiler report API and focused tests
- feature-registry linkage for P22.0
- `docs/audits/full-feature-surface-audit-v1.md`
- updates to current supported surface / explainability / truthful intent docs
- regenerated docs manifest
