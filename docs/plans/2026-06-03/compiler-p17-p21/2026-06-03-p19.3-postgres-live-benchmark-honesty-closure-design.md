# P19.3 PostgreSQL Live Benchmark Honesty Closure Design

Status: local closure design for the active Ideal Master Plan goal loop.

## Scope

This slice closes the P19.3 benchmark honesty gate by linking the source-first
PostgreSQL dry-run gate to checked local SCRAM/PostgreSQL benchmark evidence
that is accepted by the existing TechEmpower report validator.

Covered evidence:

- `p19.3_postgres_source_first` dry-run source gate for DB single query,
  DB multiple queries, DB updates, and DB fortunes rows
- DB-backed SCRAM semantic report with all six local endpoints
- DB-backed SCRAM `/db` matrix report
- DB-backed SCRAM `/queries`, `/updates`, and `/fortunes` matrix report
- validator rejection coverage for weak SCRAM metadata, command provenance,
  grid mismatches, stale or missing git head, resource snapshots, and summary
  mismatches

## Non-Claims

- no official upstream TechEmpower result
- no production database benchmark or external production deployment claim
- no P20 performance matrix or cross-language comparison
- no C++/Rust parity or measured speed comparison
- no public full source-level PostgreSQL driver API promotion
- no runtime behavior change

## Implementation Shape

Add a `live_measured_db_benchmark_honesty` evidence row to
`ProductionPostgresCoverage()`. The validator must require the three checked
local SCRAM report paths, the `validate-techempower-report` command, SCRAM
metadata, `/db` plus `/queries`/`/updates`/`/fortunes` matrix coverage, and
the source-first dry-run report path.

The row remains `evidence_only`: it promotes P19.3's benchmark honesty closure,
not a production benchmark or official result.

Update the audit, benchmark docs, feature registry, generated manifest, and
goal sidecars so the closure is visible and bounded.
