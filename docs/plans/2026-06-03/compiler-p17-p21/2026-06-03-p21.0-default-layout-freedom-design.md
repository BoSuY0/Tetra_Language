# P21.0 Default Layout Freedom Design

Status: design bridge before RED/GREEN.

## Observed Control Points

Graphify MCP and concrete inspection found the narrow layout/ABI control
points:

- `compiler/internal/layoutopt/layoutopt.go` already defines
  `PolicyForStruct`.
- `compiler/internal/layoutopt/layoutopt_test.go` already proves default
  structs allow optimizer freedoms while `repr(C)` forbids them.
- `compiler/internal/semantics/layout_repr_test.go` already proves checked
  type metadata preserves `repr(C)` and default `tetra` representation.
- `compiler/reports.go` already emits `app.layout.json`, but the current
  layout report only has a one-line policy string and does not show per-struct
  decisions.
- `compiler/internal/semantics/checker.go` and `compiler/ffi_target.go`
  already validate many `@export` ABI hazards; native aggregate target tests
  currently use default structs and target-specific aggregate C ABI diagnostics.

## Design

P21.0 should be a bounded policy/report slice, not an actual field-reordering
or layout-transforming optimizer.

The slice should add:

- layout report schema version 2 with per-struct decision rows;
- summary counts for default compiler-owned rows, `repr(C)` ABI-locked rows,
  and exported/public-ABI rows;
- row fields that show repr, ABI lock status, source field order, current field
  slots/offsets, allowed future transforms, denied transforms, and decision
  reason;
- a report validator that rejects missing rows, `repr(C)` layout freedoms,
  default structs that claim C ABI lock, placeholder text, and exported/public
  ABI rows without explicit `repr(C)`;
- a semantic diagnostic for exported default structs so public ABI / exported
  FFI aggregate exposure requires explicit `repr(C)`;
- existing target aggregate ABI diagnostics preserved by changing those tests
  to use `repr(C) struct Pair`.

## Non-Goals

This slice must not:

- reorder fields;
- remove padding;
- split hot/cold fields;
- scalar-replace user-visible layouts;
- apply AoS-to-SoA transforms;
- claim C ABI for default structs;
- claim performance or runtime behavior changes.

## Test Strategy

RED should cover:

- `@export` parameter/return using a default struct is rejected with an
  explicit `repr(C)` diagnostic;
- `repr(C)` aggregate exports still reach the target-specific aggregate C ABI
  gate;
- `buildLayoutReport` emits schema v2 rows for default and `repr(C)` structs;
- `ValidateLayoutReport` rejects fake freedoms on `repr(C)` rows and exported
  rows without explicit `repr(C)`;
- `BuildFileWithStatsOpt(... Explain: true)` writes `.layout.json` containing
  the P21.0 decision evidence.

The feature registry and docs should mention the checked report artifact and
the public ABI/exported FFI explicit repr boundary.
