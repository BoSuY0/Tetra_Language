# P20.1 Performance Blocker Reports Design

Status: implementation bridge for the active P20.1 slice.

## Observed Facts

- `GOAL.md` marks P20.1 Performance Blocker Reports as active.
- The master plan requires the compiler to tell users why code is slow.
- The required reasons are:
  - `left bounds check: missing dominance`
  - `heap allocation: escapes through return`
  - `heap allocation: unknown call`
  - `not vectorized: no noalias proof`
  - `not inlined: code-size budget`
  - `register spill: live range pressure`
  - `stack fallback: unsupported aggregate return`
  - `actor copy: borrowed data crosses boundary`
- `compiler/reports.go` already emits `app.perf.json` through
  `buildPerformanceReport()`.
- Existing `.perf.json` blocker rows are evidence-only and do not affect safe
  program semantics.
- P20.0 checked report
  `reports/benchmark-matrix-hardening-v1/benchmarks/p20-matrix-hardening-report.json`
  has 17 Tetra benchmark rows, all citing
  `reports/benchmark-matrix-hardening-v1/benchmarks/artifacts/p20-matrix-hardening.perf.json`.

## Design

Extend the existing compiler performance report rather than adding a public
optimization mode.

The P20.1 report shape:

- keeps `kind: perf`;
- promotes the report envelope to schema version 3;
- records the P20.0 matrix scope and report artifact path;
- records the exact eight master-plan blocker reasons as canonical rows;
- adds one actionable benchmark explanation row for every P20.0 Tetra
  benchmark name/category;
- keeps all claims as non-claims and policy notes.

Each blocker reason must include:

- stable code;
- exact master-plan message;
- compiler component;
- evidence artifact or report family;
- user-facing next step.

Each benchmark explanation row must include:

- P20.0 benchmark name and category;
- matrix scope and matrix report artifact;
- reason codes that exist in the canonical blocker list;
- artifact paths to the relevant compiler report families;
- an actionable explanation and next-step text.

## Validation

Add an internal validator in `compiler/reports.go` that rejects:

- missing report metadata;
- missing or duplicate blocker rows;
- missing master-plan reason messages;
- empty evidence/next-step text;
- missing P20.0 benchmark explanation rows;
- benchmark rows that cite unknown reason codes;
- benchmark rows with weak placeholder text;
- broad fastest-language, C++/Rust parity, official benchmark, official
  TechEmpower, measured-speed, or runtime-behavior-change claims.

## Evidence Artifact

Update
`reports/benchmark-matrix-hardening-v1/benchmarks/artifacts/p20-matrix-hardening.perf.json`
to the same schema so the existing P20.0 matrix has a checked performance
blocker artifact without claiming benchmark execution or measured speed.

## Non-Claims

This slice does not claim measured performance, C++/Rust parity, official
benchmark status, official TechEmpower status, P20.2 claim tiers, optimizer
behavior changes, runtime behavior changes, or complete elimination of any
blocker. It only proves actionable explanation coverage.
