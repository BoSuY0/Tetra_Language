# P22.1 First-class Callables v1 Design

Date: 2026-06-03
Slice: P22.1 First-class Callables v1

## Intent

P22.1 closes the master-plan callable requirement with same-branch evidence for
escaping callable values that keep a safe ABI. Existing implementation already
contains the core pieces:

- `compiler/internal/semantics/callable_escape.go` classifies callable escape
  boundaries.
- `compiler/internal/semantics/types.go` fixes `FnPtrEnvSlotCount = 8`,
  `FnPtrSlotCount = 9`, and `CallableHandleSlotCount = 4`.
- `compiler/internal/semantics/closure_captures.go` limits captures to safe
  immutable by-value types and rejects unsupported mutable/resource captures.
- `compiler/internal/lower/callables.go` emits the compact `fnptr` fast path
  and the larger callable handle path.
- Interface generation preserves returned callable handle metadata for
  cross-module validation.

The missing P22.1 piece is an explicit coverage report and validator that tie
these facts to the master-plan done-when and prevent future docs/manifest
drift or fake callable promotion claims.

## Report Shape

Add a compiler-level report:

- schema: `tetra.language.first_class_callables.v1`
- scope: `p22.1_first_class_callables_v1`
- rows:
  - `fnptr_fast_path`
  - `fat_callable_handle`
  - `capture_safety_classifier`
  - `mutable_capture_escape_diagnostics`
  - `resource_thread_escape_diagnostics`
  - `fixed_abi_width`
  - `cross_module_interface_metadata`
  - `storage_and_callback_paths`

The report should include a concrete witness built by parsing, checking, and
lowering representative programs:

- a bounded one-capture callable that stays within the 9-slot `fnptr` ABI and
  does not allocate a callable environment;
- a nine-capture callable that uses the 4-slot handle, allocates a heap
  environment, writes and reads all captured slots, and calls the closure with
  explicit argument plus captured environment slots.

## Validator Guardrails

The validator must reject:

- missing or duplicate rows;
- placeholder evidence;
- missing witness coverage;
- fake variable-width ABI or exploding return-slot claims;
- fake by-reference mutable capture support;
- fake pointer/resource capture support;
- fake thread-boundary callable transfer;
- fake runtime generic callable polymorphism or dynamic callable dispatch;
- fake unsafe lifetime relaxation;
- performance claims;
- runtime behavior changes outside the existing callable ABI;
- safe-program semantic changes.
