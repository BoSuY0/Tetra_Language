- P23.0 translation validation v2 records optimizer validation, allocation
  lowering validation, proof preservation, backend differential rows, and

- P23.1 fuzz/property/differential evidence records deterministic parser,
  checker, PLIR/lowering, backend matrix, allocator-property, actor-transfer,

- P23.2 formal core v1 records machine-checkable values, provenance,

- `compiler/internal/runtimeabi` records allocation contracts, region allocator

- `compiler/internal/stdlibrt.RegionAwareStdlibCoverage` records current
  region-aware stdlib evidence and explicit production-stack non-claims.

## Design

P23.3 adds a top-level compiler evidence report with schema
`tetra.self_hosting.gate.v1` and scope `p23.3_self_hosting_gate`. The report is
a promotion gate, not a promotion. It defines the current self-host subset and
then records which required self-hosting blockers remain closed.

Rows:

- self-host subset definition;
- small compiler component compile boundary;
- Go compiler output vs Tetra-compiled output comparison boundary;
- register backend stability gate;
- optimizer validation maturity gate;
- allocator/runtime stability gate;
- stdlib sufficiency gate;
- deterministic bootstrap chain gate;
- cross-platform bootstrap story gate;
- no self-hosting claim.

The report must be valid while `SelfHostingClaimed` is false and
`GateDecision.Allowed` is false. It must reject any fake report that claims
self-hosting, omits blocker rows, uses placeholder evidence, marks a blocked
row as satisfied without witness evidence, or weakens compiler subset,
backend, optimizer, allocator/runtime, stdlib, deterministic bootstrap, or
cross-platform bootstrap requirements.

## Implementation Shape

- Extend `compiler/internal/selfhostgate` so `Evidence` includes compiler
  subset, small component compile, output comparison, deterministic bootstrap,
  and cross-platform bootstrap gates in addition to the existing backend,
  optimizer, allocator, and stdlib evidence.
- Add `compiler/self_hosting_gate_v1.go` with
  `BuildP23SelfHostingGateV1Report()` and
  `ValidateP23SelfHostingGateV1Report()`.
- Reuse live evidence from P23.0/P23.1/P23.2 reports, `differential`,
  `runtimeabi`, and `stdlibrt` rather than inventing a parallel proof system.
- Link the report through the verified-track registry, audit docs,
  `current_supported_surface`, `truthful_intent_architecture`,
  `explainable_one_build`, and `docs/generated/manifest.json`.

## Non-claims

- Tetra is not self-hosting.
- No Tetra compiler component is claimed to compile itself yet.
- No Go compiler output vs Tetra-compiled output equivalence is claimed yet.
- No deterministic bootstrap chain is claimed yet.
- No cross-platform bootstrap story is claimed yet.
- Runtime behavior does not change.
- Safe-program semantics do not change.
- No performance claim is made.
