# Tetra v1.0 Release Notes Draft

Status: Draft snapshot as of 2026-04-26.

This draft records what is verified now versus what still blocks a true v1.0
release label.

## Verified In This Snapshot

- Docs manifest regenerated and validated.
- Docs verification and doctests pass.
- Build-only smoke passes for all mandatory targets: `linux-x64`, `macos-x64`,
  `windows-x64`, `wasm32-wasi`, `wasm32-web`.
- Native host smoke (`linux-x64`) runs.
- WASI smoke runs with a scripted fallback runner (`node-wasi`) and emits a
  validated smoke report.
- API diff baseline + candidate diff are generated and enforced in
  `no-change` mode.
- Reproducible build proof passes for one native target (`linux-x64`) and one
  WASM target (`wasm32-wasi`).

## Remaining Release Blockers

- Compiler/test baseline is not green for v1.0 gate commands.
- `go test ./compiler/... ./cli/... ./tools/... -count=1` fails in existing UI
  parser/semantics tests and semantic clause coverage tests.
- `bash scripts/test_all.sh --full --keep-going --report-dir /tmp/release-prep-full`
  fails because it runs the same failing go tests.
- `./tetra version` reports `v0.6.0`, not `v1.0.x`.
- No UI-specific smoke source exists in `examples/` for browser automation.
  The fallback browser smoke path works, but this does not satisfy the full UI
  release criterion.

This draft is still a pre-release tracking artifact. It must not be treated as
an announcement that v1.0 is released until `bash scripts/release_v1_0_gate.sh`
passes end-to-end on the release branch.
