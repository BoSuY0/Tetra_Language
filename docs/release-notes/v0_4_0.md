# Tetra v0.4.0 Release Notes

Status: final Linux-x64 production release with canonical memory,
parallelism, and compiler production gate evidence.

These notes describe the scoped `v0.4.0` objective: production Linux x64 first,
with EcoNet explicitly excluded. They are not cross-platform, WASM, EcoNet, or
v1.0 release notes.

## Selected Objective

`v0.4.0` is selected as a Linux-x64 production milestone. The selected language,
CLI, runtime, actor, and native UI behavior must be real, executable, tested,
documented, and release-evidenced. The release must not describe mock, fake,
placeholder, preview-only, metadata-only, or build-only behavior as production.

## Included Production Scope

- Linux-x64 compiler/CLI build and run support.
- Linux-x64 Memory Production Core evidence
  (`tetra.memory.production.v1`).
- Linux-x64 Parallelism Production Core evidence
  (`tetra.parallel.production.v1`).
- Linux-x64 Compiler Production Core evidence
  (`tetra.compiler.production.v1`).
- Callable Level 1, Callable Level 2, and selected safe first-class callable
  behavior.
- Local/control-flow lifetime SSA ownership/resource checks for the selected
  surface.
- Standard-library compatibility mirrors that forward to `lib.core.*`.
- UI metadata v1 for the selected Linux-x64 UI/native shell surface.
- Linux-x64 distributed actor runtime evidence.
- Linux-x64 native UI runtime evidence.

## Explicitly Excluded

- EcoNet / distributed Eco / hosted production TetraHub networking.
- Full v1.0 language guarantees.
- WASM production runtime claims (`wasm.runtime-execution`, `wasm32-wasi`,
  `wasm32-web`).
- Windows and macOS production runtime claims.
- Non-Linux native UI runtime/toolkit/accessibility integration claims.
