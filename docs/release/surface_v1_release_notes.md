# Tetra Surface v1 Release Notes

Status: current for `surface-v1-linux-web` after the Surface v1 release gate
passes. Headless Surface is a release evidence target.

## Current

- Headless deterministic release evidence target.

## Unsupported

- macOS real-window Surface.
- Windows real-window Surface.
- wasm32-wasi Surface UI runtime.
- GPU renderer.
- Dynamic trait-object widgets and witness-table component dispatch.
- Arbitrary native platform widgets.
- DOM/React/user-JS application UI.
- Full rich text editor or IDE-grade text editing.
- Full AT-SPI or full screen-reader parity without separate real probe
  artifacts.

## Migration

Existing `ui.metadata-v1` apps remain on the legacy metadata compatibility
surface. New release-supported Surface apps should use ordinary Tetra structs,
`lib.core.component`, `lib.core.widgets`, `lib.core.text`,
`lib.core.accessibility`, and `lib.core.style`.
Future Block System apps should use Block configuration as the main visual
material. Current Block evidence is scoped and experimental; release-supported
Surface v1 apps still use the bounded `lib.core.widgets` subset.
P32 adds `tetra.surface.migration-report.v1` /
`surface-widget-block-migration-v1` evidence from `migration-gate.sh` and
`validate-surface-migration-report`: `lib.core.widgets` stays a compatibility
layer, Panel/Button/TextBox/StatusText map to Block/Morph recipe equivalents,
existing Surface v1 widget examples still pass, and new production UI docs
recommend Block/Morph recipes. Widgets are not the core final architecture and
are not deprecated before production examples and gates cover replacement.

P33 adds `tetra.surface.example-suite-report.v1` /
`surface-production-example-suite-v1` evidence from `example-suite-gate.sh` and
`validate-surface-example-suite`: ten realistic executable
`examples/surface_prod_*.tetra` app shapes run as Block/Morph-only production
examples with scoped headless/linux-x64/wasm32-web target coverage, ecosystem
seed metadata, and guards that reject screenshot-only examples,
React/Electron/DOM runtime dependencies, widgets where Block/Morph is required,
missing app shapes, missing scoped target coverage, and toy visual-only
examples. This does not claim broad cross-platform parity.

P34 adds `tetra.surface.prod-gate-report.v1` /
`surface-production-ci-release-gate-v1` evidence from `prod-gate.sh` and
`validate-surface-release-state --scope
PROD_STABLE_SCOPED_LINUX_WEB_APP_UI`. The release workflow runs this as the
required `surface-production-final/**` artifact upload without
`continue-on-error`. It aggregates release, Block/Morph, visual, package,
security, IPC/lifecycle, crash, i18n, performance, migration, example-suite,
API stability, prod-claim governance, and artifact hash evidence; missing CI
job, skipped production target counted as pass, and missing artifact hash
manifest are rejected.

P35 adds `tetra.surface.prod-audit.v1` /
`surface-prod-final-same-commit-audit-v1` evidence in
`docs/release/surface_prod_release_audit.md`, validated by
`validate-surface-prod-audit`. The public claim now comes from the audit
verdict enum
`PROD_STABLE_SCOPED/NEAR_READY_WITH_BLOCKERS/BETA_ONLY/EXPERIMENTAL_ONLY/FAIL`;
clean checkout violations, reports from a different git head, missing
unsupported-target nonclaims, and hand-written optimistic public claims are
rejected.

P36 adds `tetra.surface.electron-comparison-report.v1` /
`surface-electron-comparison-method-v1` evidence from
`surface-electron-comparison` and
`validate-surface-electron-comparison-report`. The allowed public wording is
competitive with Electron in the supported scope. Public benchmark-superiority
claim, cherry-picked hardware, missing variance, unfair app shape, missing
environment, and single-smoke faster-than-Electron claims are rejected.

## Known Limits

Surface v1 is a bounded release, not a general Qt/Flutter/browser framework
replacement. It proves the linux-x64 real-window and wasm32-web browser-canvas
release paths plus headless evidence. Broader platform targets, richer text,
GPU rendering, dynamic widget dispatch, and full platform accessibility remain
post-release work.
