# Surface Morph Recipe Cookbook

Status: experimental P08 authoring evidence.

Morph recipes are a convenience layer over `lib.core.block`. They provide compact app-authoring
names like `form.field@1`, `nav.item@1`, and `dialog.panel@1`, but every recipe must still expand
into a `Block` graph. They do not add React, Electron, DOM UI, CSS cascade runtime, platform-native
widgets, or new core primitives.

## Recipe Contract

Required recipes:

- `control.action@1`
- `field.text@1`
- `command.item@1`
- `region.panel@1`
- `form.field@1`
- `nav.item@1`
- `metric.tile@1`
- `dialog.panel@1`
- `toast.notification@1`
- `tab.item@1`
- `list.row@1`
- `app.shell@1`
- `toolbar@1`
- `split.pane@1`
- `status.bar@1`
- `settings.form@1`
- `log.row@1`
- `empty.state@1`
- `error.panel@1`

Required affordances:

- `action`
- `field.text`
- `toggle`
- `navigation`
- `region`
- `overlay`
- `status`

Each recipe row in `tetra.surface.morph.v1` evidence declares slots and inputs, sets `output` to
`Block`, sets `expands_to_block_graph` to true, and keeps `hidden_app_state`, `platform_widgets`,
and `core_primitive_promotion` false.

## Reference Apps

The P08 reference set is:

- `examples/surface/morph_core/surface_morph_command_palette.tetra`
- `examples/surface/morph_core/surface_morph_project_dashboard.tetra`
- `examples/surface/morph_core/surface_morph_settings.tetra`
- `examples/surface/morph_core/surface_morph_editor_shell.tetra`
- `examples/surface/morph_core/surface_morph_glass_panel.tetra`
- `examples/surface/morph_core/surface_morph_studio_shell.tetra`

Each app imports `lib.core.morph`, constructs recipes, records `RecipeExpansion` values, builds a
`BlockTree`, and validates the resulting Block graph. Release evidence is generated by:

```sh
bash scripts/release/surface/morph-gate.sh \
  --report-dir reports/surface-morph/gate
```

The gate summary records `recipe_count: 19`, the recipe expansion report path, and the six reference
app sources.
